AtCoder ABC455-E 复盘：前缀差值统计与三集合容斥

发表于 2026-04-26 更新于 2026-05-17 分类于 算法题解/复盘
一道关于字符串子串计数的问题复盘：从暴力枚举子串出发，转化为前缀差值状态统计，并通过容斥处理三种字符数量相等的重复计数。

前言

本题给定一个只由 A、B、C 三种字符组成的字符串 SSS，要求统计有多少个非空子串满足：

子串中 A、B、C 的出现次数两两不同。

设某个子串中三种字符的出现次数分别为 a,b,ca,b,ca,b,c，那么合法条件为：

a,b,c 两两不同 a,b,c \text{ 两两不同} a,b,c 两两不同

等价于：

a≠b,b≠c,c≠a a\ne b,\quad b\ne c,\quad c\ne a a=b,b=c,c=a

字符串长度满足：

1≤N≤2×105 1\le N\le 2\times 10^5 1≤N≤2×105

因此不能直接枚举所有子串。

一、暴力做法的问题

最直接的想法是枚举所有子串 [l,r][l,r][l,r]，再通过前缀和 O(1)O(1)O(1) 求出其中 A、B、C 的出现次数。

例如预处理：

则子串 [l,r][l,r][l,r] 中三种字符的出现次数为：

a=preA[r]−preA[l−1] a=preA[r]-preA[l-1] a=preA[r]−preA[l−1] b=preB[r]−preB[l−1] b=preB[r]-preB[l-1] b=preB[r]−preB[l−1] c=preC[r]−preC[l−1] c=preC[r]-preC[l-1] c=preC[r]−preC[l−1]

然后判断 a,b,ca,b,ca,b,c 是否两两不同。

但是子串数量为：

N(N+1)2 \frac{N(N+1)}{2} 2N(N+1)​

当 N=2×105N=2\times 10^5N=2×105 时，数量达到 101010^{10}1010 级别，因此 O(N2)O(N^2)O(N2) 枚举子串不可行。

本题的关键不在于如何快速求一个子串的字符数量，而在于：

如何不枚举子串，却能统计满足某种数量关系的子串个数。

二、反向统计不合法子串

题目要求统计满足：

a≠b,b≠c,c≠a a\ne b,\quad b\ne c,\quad c\ne a a=b,b=c,c=a

的子串。

直接统计两两不同并不方便，因此考虑统计其补集。

不合法子串满足至少一个条件：

a=b或b=c或c=a a=b \quad \text{或} \quad b=c \quad \text{或} \quad c=a a=b或b=c或c=a

于是可以先统计：

  • A 数量等于 B 数量的子串个数
  • B 数量等于 C 数量的子串个数
  • C 数量等于 A 数量的子串个数

然后用总子串数量减去不合法数量。

但是这三个集合之间存在重叠。如果一个子串满足：

a=b=c a=b=c a=b=c

那么它会同时被计入：

  • a=ba=ba=b
  • b=cb=cb=c
  • c=ac=ac=a

也就是被重复统计三次。

而作为“不合法子串”，它本来只应该被统计一次。

因此后续需要使用容斥处理这一部分重复。

三、从 A=BA=BA=B 到前缀差值相等

先只考虑一个条件：某个子串中 A 和 B 的数量相等。

定义前缀差值：

di=preA[i]−preB[i] d_i=preA[i]-preB[i] di​=preA[i]−preB[i]

对于子串 [l,r][l,r][l,r]，若其中 A 和 B 数量相等，则有：

preA[r]−preA[l−1]=preB[r]−preB[l−1] preA[r]-preA[l-1]=preB[r]-preB[l-1] preA[r]−preA[l−1]=preB[r]−preB[l−1]

移项可得：

preA[r]−preB[r]=preA[l−1]−preB[l−1] preA[r]-preB[r]=preA[l-1]-preB[l-1] preA[r]−preB[r]=preA[l−1]−preB[l−1]

也就是：

dr=dl−1 d_r=d_{l-1} dr​=dl−1​

因此：

子串 [l,r][l,r][l,r] 中 A 和 B 数量相等，等价于前缀位置 rrr 与 l−1l-1l−1 的差值 preA−preBpreA-preBpreA−preB 相等。

这样一来，问题就从“枚举子串”转化为“统计相同前缀状态出现了多少次”。

十一、复杂度分析

扫描字符串一次，每个位置只进行常数次哈希表查询和更新，因此时间复杂度为：

O(N) O(N) O(N)

哈希表中存储的前缀状态数量最多为 O(N)O(N)O(N)，因此空间复杂度为：

O(N) O(N) O(N)

在本题中：

N≤2×105 N\le 2\times 10^5 N≤2×105

因此该复杂度可以通过。

结语

这题的关键不在于前缀和本身，而在于将“子串中的字符数量关系”转化为“两个前缀状态相等”。

对于 a=ba=ba=b 这类条件，可以用一维前缀差值统计；对于 a=b=ca=b=ca=b=c 这类同时满足两个等式的条件，则需要用二维前缀状态统计。

最后再通过容斥处理三个不合法集合之间的重复计数。

整体思路可以概括为：

  1. 用总子串数作为全集
  2. 反向统计至少两个字符数量相等的不合法子串
  3. 用前缀差值统计三个一维条件
  4. 用二维差值统计三者相等的重叠部分
  5. 通过容斥得到最终答案

这类题的启发在于：

前缀和不只可以用来快速求区间和，也可以通过“状态相等”来统计满足特定区间关系的数量。
